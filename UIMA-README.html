UIMA Indri Components

Pear files for Windows XP and linux (x68 Fedora core 3) are available from the UIMA component repository. The shared library and indri Java wrappers were built with the indri-2.4 distribution, available from The Lemur Project.

IndriCasConsumer

This component accepts an IndexBuildSpecification, enabling its use as a drop in replacement for the Semantic Search CAS Indexer. This component accepts Indri repository construction parameters, enabling direct processing of the raw document text with Indri's built in tokenization and stemming algorithms.

This CAS Consumer takes 5 parameters:

  • index - path to directory for repository
  • stemmer - optional stemmer to use. One of krovetz, porter
  • stopwords - optional file containing a list of stopwords
  • memory - optional memory bound for indexing
  • buildspec - IndexBuildSpecification url.

An example CPE that uses the uima_examples project is included. The descriptor for the CPE is in desc/TestConsumer.xml. The IndexBuildSpecification is in desc/buildspec.xml, and the IndriCasConsumer descriptor is in desc/IndriCasConsumerDescriptor.xml. Each indexBuildItem in the IndexBuildSpecification that is either a Breaking or Annotation style type will be indexed as a field by IndriCasConsumer. Field names are derived by stripping the package name, eg com.ibm.uima.tutorial.RoomNumber is indexed as roomnumber. In the example IndexBuildSpecification roomnumber, personname, and sentence annotations are indexed.

IndriSearch

To support querying an Indri repository within the UIMA framework, we have developed an SIAPI compliant query processor, suitable for use as a drop in replacement in Semantic Search applications. This component supports the Indri structured query language. The component includes an SIAPI implementation factory, SearchService, Searchable, Query, and Result interface implementations. The IndriSearch application is a modification of the UIMA example SemanticSearch application that uses the Indri Searchable. The GUI version could be modified in a similar fashion. Future work will add an Annotator for query results.

Required Resources

The PEAR files come prepackaged with indri_jni.dll (Windows) libindri_jni.so (Linux) in the bin folder. This folder must be on the CLASSPATH. The PEAR files come prepackaged with indri.jar in the lib folder. This jar must also be on the CLASSPATH.
